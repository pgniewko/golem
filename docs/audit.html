The Golem implementation is faithful to the plan on all critical scientific correctness points: scaler leakage is fixed, NaN validity masking works correctly, the gt-pyg v1.5.7 API is used properly, and the architecture matches the CALIBRATE notebook specification.

Key additions beyond plan

  • Protomer validation (_is_valid_protomer) — rejects Dimorphite-DL failure modes
  • CLI flags: --batch-size, --lr, --no-isoforms, --verbose

  • 2 extra notebooks: inspect_isoforms.ipynb, compare_experiments.ipynb

Key deviations from plan

  • max_tautomers=10 (plan said 25) — more conservative default
  • Per-method limits (max_tautomers, max_protomers) instead of a single max_isoforms cap — more flexible
  • Dimorphite-DL tool/fallback selection is hardcoded, not in config — simpler
  • Fine-tuning metrics computed in log scale, not original scale (D4 violation)
  • No fine-tuning early stopping in notebooks
  • Scheduler not saved in checkpoint (E.8 deviation)
  • Notebooks live in notebooks/ directly, not notebooks/examples/

2.1 — Winsorization & Standardization Details

Key findings:
  • Winsorization range [-6, 6] — taken directly from CheMeleon's torch.clamp(scaled, -6, 6).
  • Features ARE standardized — both CheMeleon and Golem perform per-feature z-score normalization (mean=0, std=1) before winsorizing. Golem uses np.nanmean/nanstd ignoring invalid positions, then clips to [-6, 6].
  • The Golem scaler initially had a data leakage bug (B.1.1 in the original plan — fitting on all data). This is now fixed: scaler fits on the train split only.

CheMeleon pipeline: Welford online stats on train → z-score → torch.clamp(-6, 6)

Golem pipeline: np.nanmean/nanstd on train → z-score → np.clip(-6, 6)

Golem implementation: descriptors.py:112-173 (NaNAwareStandardScaler.fit() and .transform())

2.2 — CheMeleon Architecture Notes

  • D-MPNN: BondMessagePassing d_h=2048, depth=6. RegressionFFN head (3-layer, input=2048, hidden=1024). Total ~9.3M params.
  • MLP-PLR: Separate descriptor autoencoder with PeriodicEmbeddings (48 frequencies). Uses input masking AND loss masking. Not a GNN.
  • Fine-tuning: Full fine-tuning of all parameters (no frozen layers). Task heads randomly initialized.
  • Fingerprints: 2048D embeddings extracted via MeanAggregation from the pretrained D-MPNN.

5. Current Bugs

5.1 — Critical (3)

Bug C1: metrics_file handle leak on exception

pretrain.py:403  |  CRITICAL

metrics_file = open(metrics_path, "w", newline="")   # line 403
# ... training loop runs ...
metrics_file.close()                                  # line 476

If training crashes between lines 403 and 476 (OOM, CUDA error, KeyboardInterrupt, etc.), the file handle leaks and the CSV may be incomplete or corrupted. No context manager or try/finally protects it.

Fix: Wrap in with open(...) as metrics_file: context manager, or use try/finally to ensure close.

Bug C2: epoch undefined if max_epochs=0

pretrain.py:465  |  CRITICAL

for epoch in range(config.max_epochs):    # line 421 — never executes if max_epochs=0
    ...
# After loop:
_save_checkpoint(..., epoch=epoch, ...)   # line 465 — NameError: 'epoch' is not defined

If config.max_epochs is 0, the for-loop body never executes and epoch is unbound, causing a NameError.

Fix: Initialize epoch = 0 before the loop (line ~420).

Bug C3: _setup_logging destroys root handlers

pretrain.py:60-61  |  CRITICAL

for h in root.handlers[:]:
    root.removeHandler(h)

If pretrain() is called from a notebook or library context with existing logging configuration (Jupyter, application logger, etc.), those handlers are silently destroyed. Fine for CLI-only usage, but a footgun for programmatic callers.

Fix: Use a named logger (logging.getLogger("golem")) with propagate=False instead of reconfiguring the root logger.

5.2 — Medium (4)

Bug M1: YAML !!python/tuple tags in resolved_config.yaml

pretrain.py:256  |  MEDIUM

yaml.dump(asdict(config), f, default_flow_style=False, sort_keys=False)

dataclasses.asdict() preserves Python tuple types. yaml.dump() with the default Dumper serializes them as !!python/tuple. Loading with yaml.safe_load() will fail. The report module already works around this by using yaml.FullLoader.

# Resulting YAML:
ph_range: !!python/tuple
- 6.4
- 8.4

Fix: Convert tuples to lists before dumping: yaml.dump(json.loads(json.dumps(asdict(config))), ...) or use a custom representer.

Bug M2: No training resume capability

cli.py, pretrain.py  |  MEDIUM

last_checkpoint.pt is saved (line 462-474) but there is no --resume CLI flag. A 500-epoch pretraining run that crashes at epoch 400 must restart from scratch. The checkpoint contains optimizer state but not scheduler state (see deviation 3.3 above).

Bug M3: Scheduler not in checkpoint

pretrain.py:527-530  |  MEDIUM

Plan E.8 specifies scheduler=scheduler in save_checkpoint(). The actual _save_checkpoint function does not pass the scheduler. Even if resume were implemented, the LR schedule would restart from epoch 0.

Additionally, the LR schedule is manual (_warmup_cosine_lr writes directly to param_groups) rather than using PyTorch's LRScheduler, making it inherently non-serializable.

Bug M4: No descriptor caching

pretrain.py:318-321  |  MEDIUM

Mordred descriptor computation is the slowest step (minutes for thousands of molecules). If the pipeline crashes after descriptors are computed but before training, all computation is lost. No caching to disk.

Recommendation: Save descriptor matrix + validity mask to a .npz file after computation, check for it on startup.

8. Code Quality Positives

  • Config layering (defaults ← YAML ← CLI) is clean and the nested YAML flattening logic handles edge cases well.
  • Validity mask approach is genuinely better than CheMeleon's — the plan's assessment is correct. NaN positions never enter the loss.
  • Error handling in isoforms is thorough — every RDKit/Dimorphite call is try-wrapped with debug logging and graceful fallbacks.
  • Scaler serialization is solid, with round-trip tests (state_dict → from_state_dict) and proper edge-case coverage (constant columns, all-NaN columns).
  • Notebook structural identity is maintained — all 4 fine-tuning notebooks share identical code after the config cell.
  • Pretrain logs confirm the pipeline works end-to-end (test runs show descriptor computation, training, checkpoint saving, report generation).
  • Masking implementation correctly handles the edge case where final_mask.sum() == 0 with a fallback to all valid positions, and skips batches with no valid positions at all.
  • Protonation fallback chain (Dimorphite-DL → RDKit Uncharger) is exactly what the plan recommends and handles ImportError gracefully.
  • Report generator (report.py) produces a self-contained Chart.js dashboard with no external dependencies beyond the CDN script — a useful addition beyond the plan.
