1. Overview

Golem pretrains a gt-pyg GraphTransformerNet backbone using Masked Descriptor AutoEncoding (MDAE) on Mordred 2D molecular descriptors. The pretrained weights transfer to downstream property-prediction tasks via fine-tuning.

Inspired by CheMeleon, with key improvements:

  • NaN-aware validity masking — invalid descriptor positions are excluded from the loss entirely, rather than filled with zeros
  • Isoform enumeration — tautomers, protonation states, and neutralized forms expand the training set (~7.4× on average)
  • Train-only scaler — standardization statistics computed exclusively on the training split (no data leakage)
  • Descriptor caching — Mordred computations cached to disk, keyed by SMILES content hash
  • Training resume — checkpoint includes optimizer and LR scheduler state for seamless resumption

2.1 — Winsorization & Standardization Details

Key findings:
  • Winsorization range [-6, 6] — taken directly from CheMeleon's torch.clamp(scaled, -6, 6).
  • Features ARE standardized — both CheMeleon and Golem perform per-feature z-score normalization (mean=0, std=1) before winsorizing. Golem uses np.nanmean/nanstd ignoring invalid positions, then clips to [-6, 6].
  • The scaler fits on the train split only to prevent data leakage.

CheMeleon pipeline: Welford online stats on train → z-score → torch.clamp(-6, 6)

Golem pipeline: np.nanmean/nanstd on train → z-score → np.clip(-6, 6)

Golem implementation: descriptors.py (NaNAwareStandardScaler.fit() and .transform())

2.2 — CheMeleon Architecture Notes

  • D-MPNN: BondMessagePassing d_h=2048, depth=6. RegressionFFN head (3-layer, input=2048, hidden=1024). Total ~9.3M params.
  • MLP-PLR: Separate descriptor autoencoder with PeriodicEmbeddings (48 frequencies). Uses input masking AND loss masking. Not a GNN.
  • Fine-tuning: Full fine-tuning of all parameters (no frozen layers). Task heads randomly initialized.
  • Fingerprints: 2048D embeddings extracted via MeanAggregation from the pretrained D-MPNN.

3. Pretraining Pipeline

The full pipeline is orchestrated by pretrain.py:pretrain():

  1. Load SMILES from .smi or .csv (auto-detected by extension)
  2. Enumerate isoforms (optional) — tautomers, protonation states, neutralized forms via Gypsum-DL/RDKit
  3. Filter invalid SMILES — RDKit sanitization check before descriptor computation
  4. Compute Mordred 2D descriptors with disk caching (SHA-256 keyed .npz)
  5. Split into train / val / test (70/20/10) before fitting scaler
  6. Fit scaler on train only (NaNAwareStandardScaler)
  7. Transform + winsorize all splits
  8. Build PyG datasets via get_tensor_data with descriptor targets in data.y
  9. Train with masked MSE (15% random mask ∩ validity mask), linear warmup + cosine decay LR, early stopping
  10. Save checkpoints (best + last) with model, optimizer, scheduler, scaler, descriptor names, config, split indices
  11. Evaluate test RMSE on best model
  12. Generate HTML report (Chart.js dashboard)

3.1 — Masking Strategy

Each training batch applies a 15% random mask intersected with the validity mask:

rand_mask = (torch.rand_like(targets) < 0.15).bool()
final_mask = rand_mask & valid_mask           # only valid + randomly selected
loss = F.mse_loss(pred[final_mask], targets[final_mask])

If no positions survive the intersection (extremely rare), the loss falls back to all valid positions. Batches with zero valid positions are skipped entirely.

3.2 — LR Schedule

Linear warmup for warmup_epochs (default 25), then cosine decay to near-zero. Implemented as a torch.optim.lr_scheduler.LambdaLR for serialization into checkpoints.

3.3 — Descriptor Caching

Mordred computation (the slowest step) is cached to {output_dir}/descriptors_{hash}.npz. The hash is a 16-character SHA-256 of the ordered SMILES list. Subsequent runs with identical SMILES skip recomputation.

3.4 — Training Resume

The --resume CLI flag loads model weights, optimizer state, and scheduler state from an existing checkpoint. Training continues from the saved epoch with the correct LR position.

4. Architecture & Design Decisions

4.1 — Config Layering

Configuration merges three layers: defaults (dataclass fields) ← YAML (configs/pretrain_openadmet.yaml) ← CLI overrides. Each layer can override the previous. Nested YAML structures (e.g., isoforms.tautomers.enabled) are flattened automatically.

4.2 — Isoform Enumeration

Three methods expand the molecular dataset:

  • Tautomers — RDKit TautomerEnumerator (max 10 per molecule)
  • Protonation — Gypsum-DL/Dimorphite-DL primary, RDKit Uncharger fallback (pH 6.4–8.4, max 10 protomers)
  • Neutralization — RDKit Uncharger

All isoforms are canonicalized, validated (with protomer rejection for known failure modes like NH4+ on organic N), and deduplicated. The original SMILES is always at index 0.

The built-in mu_mlp head serves as the descriptor prediction head. No wrapper model is needed — GraphTransformerNet is used directly with zero_var=True for pretraining.

4.4 — Checkpoint Format

Checkpoints (via gt-pyg's save_checkpoint) contain:

  • Model state dict
  • Optimizer state dict
  • Scheduler state dict
  • Epoch number and best validation metric
  • Scaler state (means, stds, winsorize range)
  • Descriptor names and count
  • Full resolved config
  • Train/val/test split indices

4.5 — NaN-Aware Standard Scaler

NaNAwareStandardScaler computes per-feature statistics using only valid (non-NaN, finite) values. Features with all-NaN columns get std forced to 1.0 (no scaling). After z-score normalization, values are clipped to [-6, 6].

The scaler supports round-trip serialization (state_dict / from_state_dict) for checkpoint storage.

7. Code Quality Highlights

  • Config layering (defaults ← YAML ← CLI) is clean and the nested YAML flattening logic handles edge cases well.
  • Validity mask approach is genuinely better than CheMeleon's — NaN positions never enter the loss.
  • Error handling in isoforms is thorough — every RDKit/Gypsum-DL call is try-wrapped with debug logging and graceful fallbacks.
  • Scaler serialization is solid, with round-trip tests and proper edge-case coverage (constant columns, all-NaN columns).
  • Masking implementation correctly handles the edge case where final_mask.sum() == 0 with a fallback to all valid positions.
  • Protonation fallback chain (Gypsum-DL/Dimorphite-DL → RDKit Uncharger) handles ImportError gracefully.
  • Report generator produces a self-contained Chart.js dashboard with no external dependencies beyond the CDN script.
  • Named logger (golem) with propagate=False avoids destroying root handlers in notebook/library contexts.
